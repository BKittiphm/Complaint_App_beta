# รายละเอียดคุณลักษณะ (Specification) - Complaint App Beta

ไฟล์นี้ใช้เพื่อรวบรวมฟีเจอร์และโครงสร้างข้อมูลที่ต้องการสำหรับแอปแจ้งเรื่องร้องเรียน

## 📋 โครงสร้างข้อมูลพื้นฐาน (Data Schema)
*ฟิลด์ข้อมูลที่คาดว่าจะเก็บในอนาคต:*

| ชื่อฟิลด์ | ประเภทข้อมูล | คำอธิบาย |
| :--- | :--- | :--- |
| `id` | String (UUID) | รหัสอ้างอิงของเรื่องร้องเรียน |
| `title` | String | หัวข้อเรื่องที่แจ้ง |
| `description` | Text | รายละเอียดเนื้อหาของปัญหา |
| `category` | Enum | ประเภท (เช่น บริการ, สถานที่, ความปลอดภัย) |
| `status` | Enum | สถานะ (รอดำเนินการ, กำลังแก้ไข, เสร็จสิ้น) |
| `created_at` | DateTime | วันเวลาที่เริ่มแจ้งเรื่อง |
| `image_url` | String (URL) | ลิ้งก์รูปภาพประกอบ (ถ้ามี) |
| `location` | Object | พิกัด GPS (Latitude, Longitude) |
| `reporter_name` | String | ชื่อผู้แจ้งเรื่อง |

---

## 🚀 ฟีเจอร์ที่ต้องการ (Planned Features)

### 1. หน้าแจ้งเรื่อง (Create Complaint)
- [ ] ฟอร์มกรอกข้อมูลตาม Schema ด้านบน
- [ ] ปุ่มสำหรับเปิดกล้องถ่ายรูปประกอบ
- [ ] ระบบดึงพิกัด GPS อัตโนมัติ (Geolocation)

### 2. หน้าติดตามสถานะ (List & Track)
- [ ] แสดงรายการเรื่องที่เคยแจ้งไว้ในรูปแบบ Card
- [ ] มีตัวเลือกกรอง (Filter) ตามสถานะ
- [ ] ระบบแจ้งเตือน (Push Notification) เมื่อสถานะเปลี่ยน

### 3. ระบบจัดการ (Admin Dashboard)
- [ ] หน้าจอสำหรับเจ้าหน้าที่ในการกดเปลี่ยนสถานะเรื่อง
- [ ] ระบบสรุปยอดสถิติ (Chart) รายสัปดาห์/รายเดือน

---

## 🔒 แผนการจัดการระบบความปลอดภัย (Security & Logic)
- ผู้แจ้งสามารถดูได้เฉพาะเรื่องของตัวเอง (ในเบื้องต้น)
- ข้อมูลควรถูกเข้ารหัสก่อนส่งขึ้นคลาวด์
- รองรับการใช้งาน Offline (บันทึกข้อมูลไว้ในเครื่องก่อนเมื่อไม่มีเน็ต)
